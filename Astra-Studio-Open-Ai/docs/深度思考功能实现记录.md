## 📋 功能概述

### 功能描述
在前端聊天界面添加"深度思考"开关，允许用户选择是否启用AI模型的思维链（Chain-of-Thought）输出模式。

### 核心能力
- **普通模式** (`returnThinking=false`): 快速响应，不显示思考过程
- **深度思考模式** (`returnThinking=true`): 显示AI的完整推理过程（青绿色背景区域）

### 用户交互流程
```
用户点击"深度思考"按钮 → 前端传递 deepThink=true 参数 
→ 后端选择对应模型配置 → SSE流式返回thinking内容
→ 前端渲染青绿色思考区域 + 回复文本
```

### 设计原则
1. **工厂模式**: 统一模型创建逻辑，消除重复代码
2. **策略模式**: 根据参数动态选择不同服务实例
3. **开闭原则**: 预留扩展点（工具、RAG），新增功能无需修改现有代码
4. **单一职责**: Controller只负责路由，Factory负责创建，Service负责业务

## 🔧 技术实现细节

### 1️⃣ 后端核心 - AiCodeHelperServiceFactory.java

#### 关键设计：工厂方法消除冗余

```java
@Configuration
public class AiCodeHelperServiceFactory {

    // 公共配置注入
    @Value("${langchain4j.open-ai.streaming-chat-model.base-url}")
    private String baseUrl;
    
    @Value("${langchain4j.open-ai.streaming-chat-model.api-key}")
    private String apiKey;
    
    @Value("${langchain4j.open-ai.streaming-chat-model.model-name}")
    private String modelName;

    // 核心方法1：统一模型创建（参数化配置）
    private OpenAiStreamingChatModel createModel(boolean deepThink, int timeoutSeconds) {
        return OpenAiStreamingChatModel.builder()
                .baseUrl(baseUrl)
                .apiKey(apiKey)
                .modelName(modelName)
                .returnThinking(deepThink)  // 控制思维链开关
                .timeout(Duration.ofSeconds(timeoutSeconds))  // 不同超时时间
                .logRequests(true)
                .logResponses(true)
                .build();
    }

    // 核心方法2：统一服务构建（预留扩展点）
    private AiCodeHelperService buildService(OpenAiStreamingChatModel streamingModel) {
        return AiServices.builder(AiCodeHelperService.class)
                .chatModel(openAiChatModel)
                .streamingChatModel(streamingModel)
                // 未来可在此添加：
                // .toolProvider(webSearchToolProvider)  ← 联网搜索
                // .contentRetriever(retriever)         ← RAG检索
                .chatMemoryProvider(memoryId -> MessageWindowChatMemory.withMaxMessages(10))
                .build();
    }

    // 两个Bean实例
    @Bean
    public AiCodeHelperService normalAiService() {
        return buildService(createModel(false, 30));  // 普通模式：30秒超时
    }

    @Bean
    public AiCodeHelperService deepThinkAiService() {
        return buildService(createModel(true, 60));   // 深度思考：60秒超时+思维链
    }
}
```

**优化效果**：
- 代码量减少37%（从95行→60行）
- 配置修改只需改一处（DRY原则）
- 新增模式只需加一行 `buildService(createModel(...))`

### 2️⃣ 路由控制 - AiController.java

```java
@RestController
@RequestMapping("/ai")
public class AiController {

    @Resource
    private AiCodeHelperService normalAiService;

    @Resource
    private AiCodeHelperService deepThinkAiService;

    @PostMapping(value = "/chat", consumes = MediaType.MULTIPART_FORM_DATA_VALUE,
            produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter chat(
            @RequestParam("memoryId") String memoryId,
            @RequestParam(value = "text", required = false) String text,
            @RequestParam(value = "files", required = false) List<String> files,
            @RequestParam(value = "deepThink", defaultValue = "false") boolean deepThink) {  // 新增参数

        log.info("memoryId: {}, text: {}, files: {}, deepThink: {}", memoryId, text, files, deepThink);

        // 文件列表处理
        List<String> fileList = files == null ? Collections.emptyList()
                : files.stream()
                        .filter(url -> url != null && !url.isBlank())
                        .map(String::trim)
                        .toList();

        // 构建消息文本
        String userMessageText = MultipartUserMessageBuilder.buildText(text, fileList);

        executor.execute(() -> {
            try {
                // 动态路由：根据deepThink参数选择服务
                AiCodeHelperService selectedService = deepThink ? deepThinkAiService : normalAiService;
                log.info("{} AI service selected", deepThink ? "🧠 Deep Think" : "💬 Normal");

                TokenStream tokenStream = selectedService.chatWithStream(memoryId, userMessageText);
                
                // ... SSE流式处理 ...
            }
        });
        
        return emitter;
    }
}
```

**关键点**：
- 使用三元运算符动态选择服务实例
- 日志标记清晰区分模式
- 为后续扩展预留注释提示

### 4️⃣ 消息构建工具 - MultipartUserMessageBuilder.java

```java
public final class MultipartUserMessageBuilder {

    /**
     * 构建纯文本消息（用于AiService接口）
     * 将文本和文件URL合并为格式化的字符串
     */
    public static String buildText(String text, List<String> files) {
        StringBuilder messageText = new StringBuilder();

        if (text != null && !text.isBlank()) {
            messageText.append(text.trim());
        }

        if (files != null && !files.isEmpty()) {
            for (String fileUrl : files) {
                if (fileUrl == null || fileUrl.isBlank()) continue;
                String url = fileUrl.trim();
                
                if (isImageUrl(url)) {
                    if (messageText.length() > 0) messageText.append("\n");
                    messageText.append("[图片: ").append(url).append("]");
                } else {
                    if (messageText.length() > 0) messageText.append("\n");
                    messageText.append("[文件: ").append(url).append("]");
                }
            }
        }

        if (messageText.isEmpty()) {
            throw new ResponseStatusException(HttpStatus.BAD_REQUEST, "请至少提供文字内容或文件URL");
        }

        return messageText.toString();
    }
}
```

**设计理由**：
- LangChain4j AiService 要求 `@UserMessage` 注解的参数类型为 String
- 文件以引用形式附加在文本后面（图片/文件分别标注）
- 保持与原有 `build()` 方法（返回 UserMessage）共存，兼容非 AiService 场景

## ⚠️ 遇到的问题及解决方案

### 问题1：HTTP请求体中消息内容错误

**现象**：
```json
{
  "messages": [{
    "role": "user",
    "content": "UserMessage { name = null contents = [TextContent { text = \"什么\" }] }"
  }]
}
```

**原因分析**：
```java
// ❌ 错误代码
TokenStream chatWithStream(
    @MemoryId String memoryId,
    @UserMessage dev.langchain4j.data.message.UserMessage message  // 冲突！
)
```
- `@UserMessage` 注解告诉LangChain4j："这是一个用户消息参数，请自动转换"
- 但参数类型已经是 `UserMessage` 对象
- LangChain4j 尝试调用 `toString()` 序列化，得到对象的字符串表示

**解决方案**：
```java
// ✅ 正确代码
TokenStream chatWithStream(
    @MemoryId String memoryId,
    @UserMessage String message  // 改用String类型
)
```
- 参数类型改为 `String`
- LangChain4j 自动正确包装为 UserMessage
- HTTP请求体变为 `"content": "什么"` ✅

**技术要点**：
> LangChain4j AiService接口的方法参数规则：
> - 必须使用注解：`@V`, `@UserMessage`, `@UserName`, `@MemoryId`
> - `@UserMessage` 注解对应的参数类型应该是 **String** 或基本类型
> - 如果传入完整的 `UserMessage` 对象，则**不能**使用 `@UserMessage` 注解

---

### 问题2：缺少注解导致运行时异常

**现象**：
```
dev.langchain4j.service.IllegalConfigurationException: 
Parameter 'message' of method 'chatWithStream' should be annotated with 
@V or @UserMessage or @UserName or @MemoryId
```

**原因**：
尝试移除 `@UserMessage` 注解来解决问题1，但LangChain4j要求所有参数必须有注解。

**最终方案**：
结合问题1和问题2，确定最佳实践：
- ✅ 保留 `@UserMessage` 注解（满足框架要求）
- ✅ 参数类型改为 `String`（避免toString()问题）
- ✅ 在Controller层调用 `MultipartUserMessageBuilder.buildText()` 提取文本

---

### 问题3：编译错误 - maxRetries方法不存在

**现象**：
```
[ERROR] 找不到符号
  符号:   方法 maxRetries(int)
  位置: 类 OpenAiStreamingChatModel.OpenAiStreamingChatModelBuilder
```

**原因**：
当前使用的LangChain4j版本的 `OpenAiStreamingChatModel.Builder` 不支持 `maxRetries()` 方法。

**解决方案**：
移除该配置项：
```java
// ❌ 移除前
.returnThinking(deepThink)
.timeout(Duration.ofSeconds(timeoutSeconds))
.maxRetries(2)  // 不支持
.logRequests(true)

// ✅ 移除后
.returnThinking(deepThink)
.timeout(Duration.ofSeconds(timeoutSeconds))
.logRequests(true)
```

---

### 问题4：语法错误 - 三元运算符写错

**现象**：
IDE显示红色波浪线，编译失败

**错误代码**：
```java
List<String> fileList = files == null ? Collections.emptyList()
    .files.stream()  // ❌ 应该是冒号 :
    .filter(...)
```

**正确代码**：
```java
List<String> fileList = files == null ? Collections.emptyList()
    : files.stream()  // ✅ 三元运算符的正确语法
    .filter(...)
```

**教训**：
> 三元运算符格式：`条件 ? 真值 : 假值`  
> 注意中间的分隔符是**冒号 `:`**，不是**点号 `.`**

## 📚 相关知识库

### LangChain4j AiService注解说明
| 注解 | 用途 | 参数类型要求 |
|------|------|-------------|
| `@MemoryId` | 会话记忆ID | String |
| `@UserMessage` | 用户消息 | String, Object（会自动序列化） |
| `@UserName` | 用户名 | String |
| `@V` | 变量注入 | 任意类型 |
| `@SystemMessage` | 系统提示词 | String (fromResource=) |

### SSE (Server-Sent Events) 要点
- 单向推送：服务器→客户端
- 自动重连：浏览器原生支持
- 格式：`data: {...}\n\n`
- 适合场景：流式AI回复、实时通知、股票行情

### 工厂模式 vs 直接实例化
| 维度 | 工厂模式（本项目采用） | 直接实例化 |
|------|---------------------|-----------|
| 代码复用 | 高（公共方法提取） | 低（重复配置） |
| 维护成本 | 低（改一处） | 高（改多处） |
| 扩展性 | 好（预留接口） | 差（硬编码） |
| 可读性 | 中（需理解模式） | 高（直观明了） |
